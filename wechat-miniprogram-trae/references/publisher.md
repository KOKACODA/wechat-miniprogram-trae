## 发布检查清单

Trae SOLO 模式按以下顺序自动执行检查，任一项不通过则暂停并提示。

### ✅ 第 1 步：合规自检

| 检查项 | 说明 | 不通过处理 |
|--------|------|-----------|
| 隐私协议 | `app.json` 配置 `__usePrivacyCheck__: true` | 自动添加配置 |
| 用户信息声明 | 调用隐私 API 前有弹窗说明 | 提示手动补充 |
| 权限说明 | `app.json` → `permission` 配置完整 | 自动补充缺失权限 |
| 营业执照/资质 | 特殊类目需上传资质 | 提示前往后台配置 |

**Trae 自动检测：**
```javascript
// 扫描 app.json 检查以下配置
const checks = [
  { key: '__usePrivacyCheck__', required: true },
  { key: 'requiredPrivateInfos', desc: '隐私接口声明' },
  { key: 'permission', desc: '权限说明' }
]
```

### ✅ 第 2 步：域名配置

| 检查项 | 说明 |
|--------|------|
| request 合法域名 | 所有 `request` / `get` / `post` 调用的域名 |
| uploadFile 合法域名 | 文件上传域名 |
| downloadFile 合法域名 | 文件下载域名 |
| socket 合法域名 | WebSocket 域名 |

**Trae 自动检测：**
1. 扫描 `utils/request.js` 中的 `BASE_URL`
2. 扫描所有 `.js` 文件中的硬编码 URL
3. 提取所有域名，与后台配置对比

```javascript
// 提取代码中所有域名
const urlPattern = /https?:\/\/([a-zA-Z0-9.-]+)/g
// 扫描所有 .js 和 .wxml 文件
```

**⚠️ 域名配置需在微信公众平台 → 开发管理 → 开发设置 → 服务器域名 中手动配置。Trae 输出需配置的域名列表。**

## 版本号管理

### 语义化版本号规范

```
主版本号.次版本号.修订号

主版本号（Major）：不兼容的 API 修改
次版本号（Minor）：向下兼容的功能新增
修订号（Patch）：向下兼容的问题修复
```

### 版本号递增规则

| 变更类型 | 版本号变化 | 示例 |
|---------|-----------|------|
| Bug 修复 | Patch +1 | 1.0.0 → 1.0.1 |
| 新增功能 | Minor +1 | 1.0.0 → 1.1.0 |
| 重大重构 | Major +1 | 1.0.0 → 2.0.0 |

### Trae 自动递增

```bash
# 读取当前版本号
# 从 project.config.json 或 git tag 获取

# 根据变更内容自动递增
# git log --oneline --since="last tag" 分析 commit message
# feat: → Minor +1
# fix: → Patch +1
# breaking: → Major +1
```

## 代码上传

### 前置条件

1. 微信开发者工具 CLI 已安装且已登录
2. 项目已在开发者工具中打开过

### 上传命令

```bash
# macOS
/Applications/wechatwebdevtools.app/Contents/MacOS/cli upload \
  --project /path/to/project \
  --version 1.0.0 \
  --desc "版本描述"

# Windows
"%ProgramFiles(x86)%\Tencent\微信web开发者工具\cli.bat" upload \
  --project C:\path\to\project \
  --version 1.0.0 \
  --desc "版本描述"
```

### Trae 自动执行

1. 读取当前版本号
2. 分析变更内容，生成版本描述
3. 执行上传命令
4. 等待上传完成，输出结果

**版本描述模板：**
```
## v{version} ({date})

### 🆕 新增
- {功能描述}

### 🐛 修复
- {修复描述}

### ♻️ 优化
- {优化描述}
```

## 体验版测试

### 生成体验版

上传代码后，在微信公众平台 → 版本管理中设置体验版。

### Trae 辅助测试清单

| 测试项 | 检查内容 |
|--------|---------|
| 首页加载 | 白屏时间 < 3s，骨架屏正常显示 |
| 页面跳转 | 所有导航路径正确，无 404 |
| 数据展示 | 列表/详情数据正确，空状态正常 |
| 表单提交 | 输入校验、提交成功/失败提示 |
| 登录流程 | 授权 → 登录 → token 刷新 |
| 支付流程 | 下单 → 支付 → 回调（如有） |
| 分享功能 | 分享卡片标题/图片/路径正确 |
| 异常处理 | 网络断开、token 过期、服务器错误 |
| 多机型 | iPhone / Android / 不同屏幕尺寸 |
| 弱网环境 | 2G/3G 网络下的加载表现 |

### 测试账号

在提交审核时需提供测试账号（如需登录的小程序）：
- 账号：`test_xxx`
- 密码：`123456`
- 确保测试账号可正常体验所有功能

## 提交审核

### 审核信息准备

| 信息 | 说明 | 注意事项 |
|------|------|---------|
| 服务类目 | 小程序功能对应的类目 | 需与实际功能一致 |
| 功能页面截图 | 核心功能页面截图 | 至少 2 张，最多 10 张 |
| 测试账号 | 需登录的小程序必须提供 | 账号可正常使用 |
| 版本描述 | 功能说明 | 简洁明了，说明核心功能 |

### Trae 辅助准备

1. 自动截图核心页面（开发者工具截图功能）
2. 生成功能描述文案
3. 检查类目配置是否与功能匹配

### 提交审核操作

```bash
# 微信开发者工具 CLI 暂不支持直接提交审核
# 需在微信公众平台手动操作：版本管理 → 提交审核
```

## 常见审核被拒及修复方案

### 🔴 类目与功能不符

**被拒原因：** 小程序类目与实际功能不匹配

**修复方案：**
1. 在微信公众平台 → 设置 → 服务类目中添加正确类目
2. 确保所有功能都在已选类目范围内
3. 如功能跨类目，需添加多个类目

**Trae 检测：** 分析 `app.json` 的 `pages` 和页面功能，对比已配置类目

---

### 🔴 隐私合规问题

**被拒原因：** 收集用户信息未说明用途 / 未配置隐私协议

| 具体问题 | 修复方案 |
|---------|---------|
| 未配置隐私协议弹窗 | `app.json` 添加 `__usePrivacyCheck__: true`，实现隐私弹窗 |
| 收集位置未说明用途 | `permission.scope.userLocation.desc` 填写具体用途 |
| 收集手机号未说明 | 按钮文案说明用途（如"用于登录"） |
| 未提供注销账号功能 | 添加账号注销入口和流程 |
| 收集信息超出必要范围 | 只收集业务必需的信息 |

**Trae 检测：** 扫描隐私 API 调用和 `app.json` 配置

---

### 🔴 虚拟支付问题

**被拒原因：** iOS 端包含虚拟商品购买功能

**修复方案：**
```javascript
// 检测平台，iOS 隐藏虚拟支付入口
const systemInfo = wx.getSystemInfoSync()
const isIOS = systemInfo.platform === 'ios'

// 页面中条件渲染
// <view wx:if="{{!isIOS}}">购买</view>
```

**Trae 检测：** 扫描支付相关代码，检查是否有平台判断

---

### 🔴 内容违规

**被拒原因：** 含有违规内容或未接入内容安全检测

**修复方案：**
1. 用户生成内容（UGC）必须接入内容安全检测
2. 调用 `wx.security.msgSecCheck` / `wx.security.imgSecCheck`
3. 配置内容审核机制

```javascript
/**
 * 文本内容安全检测
 * @param {string} content - 待检测文本
 * @returns {Promise<boolean>} 是否安全
 */
async function checkTextSecurity(content) {
  try {
    const res = await wx.cloud.callFunction({
      name: 'msgSecCheck',
      data: { content }
    })
    return res.result.errCode === 0
  } catch (err) {
    console.error('内容安全检测失败:', err)
    return false
  }
}
```

**Trae 检测：** 扫描 UGC 输入框（textarea / input），检查是否调用安全检测

---

### 🔴 诱导关注/分享

**被拒原因：** 强制或诱导用户关注公众号/分享

**修复方案：**
1. 移除"关注后查看""分享后解锁"等强制逻辑
2. 分享按钮不能作为功能使用前提
3. 不使用 `wx.onShareAppMessage` 强制触发分享

---

### 🔴 功能不完整

**被拒原因：** 审核员无法体验完整功能

**修复方案：**
1. 提供可正常使用的测试账号
2. 确保测试账号拥有完整权限
3. 在版本描述中说明测试流程
4. 核心功能无前置条件限制

---

### 🔴 页面白屏/崩溃

**被拒原因：** 审核时页面无法正常显示

**修复方案：**
1. 检查所有页面是否有加载态和错误态
2. 确保弱网环境不会白屏
3. 检查异步数据加载是否有兜底逻辑
4. 确保无 JS 运行时错误

---

### 🔴 未授权使用商标/知识产权

**被拒原因：** 使用了未授权的品牌名称/Logo/内容

**修复方案：**
1. 移除未授权的品牌元素
2. 使用原创或已授权的内容
3. 添加免责声明（如适用）

---

### 审核被拒处理流程

```
收到驳回通知
   │
   ├── 1. 阅读驳回原因
   │
   ├── 2. 对照上方修复方案定位问题
   │
   ├── 3. Trae 自动检测同类问题
   │
   ├── 4. 修复代码 → 重新上传 → 重新提审
   │
   └── 5. 如对驳回有异议 → 在公众平台发起申诉
```

## 发布后监控

### 关键指标

| 指标 | 监控方式 | 告警阈值 |
|------|---------|---------|
| 崩溃率 | 小程序数据助手 | > 1% |
| 启动耗时 | 性能监控 | > 3s |
| API 错误率 | 后端日志 | > 5% |
| 用户反馈 | 客服消息 | 差评增多 |

### 紧急回滚

如发布后出现严重问题：
1. 在微信公众平台 → 版本管理 → 回退到上一版本
2. 修复问题后重新上传发布
